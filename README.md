## Описание проекта

Веб-приложение, отображающее график цены **BTC/USD** в формате свечей (candlestick) с использованием библиотеки **TradingView Lightweight Charts™ v5**. Основная задача - реализация кастомного интерактивного элемента в виде плагина:

- **Кнопка «Add Price Level»** на правой шкале цены, появляющаяся при приближении курсора.
- **Горизонтальные линии** с ценовыми метками, создаваемые по клику.
- **Drag-and-drop** перемещение линий с полупрозрачной интерактивной зоной.
- **Удаление линий** через кнопку-крестик, появляющуюся при наведении.

Все интерактивные элементы реализованы исключительно через **Plugin API** lightweight-charts (IPrimitivePaneRenderer, IPrimitivePaneView, priceAxisPaneViews, requestUpdate), без сторонних фреймворков.

## Архитектура плагинов

lightweight-charts v5 предоставляет механизм **Custom Primitive Plugins** (ранее известный как Drawing Primitives). Каждый плагин - это класс, реализующий контракт:

- attached({ chart, series, requestUpdate }) - инициализация, подписка на события.
- detached() - очистка слушателей, предотвращение утечек памяти.
- paneViews() / priceAxisPaneViews() - возврат массива view для отрисовки на основном canvas и на canvas шкалы цены.
- updateAllViews() - обновление координат перед перерисовкой.

### AddLevelButtonPlugin

**Назначение:** отображение кнопки «+» с текущей ценой на правой шкале цены.

**Механика:** 1. Слушает mousemove на контейнере графика. 2. Если курсор находится в пределах **50 px** от правого края (chartWidth - 50), вычисляет цену через series.coordinateToPrice(y). 3. Динамически подбирает ширину плашки под текст цены (измерение через &lt;canvas&gt; контекст). 4. Через priceAxisPaneViews() отдаёт renderer, который рисует на canvas шкалы цены: - Фиолетовый скруглённый прямоугольник (#7b61ff). - Белый плюсик. - Текст цены (например, «50 123,45»). 5. При mousedown внутри геометрии кнопки вызывает колбэк onClick(price), передавая точную цену.

**Особенности hit-test:** - Проверка попадания курсора внутрь кнопки выполняется в CSS-пикселях (не bitmap), что гарантирует точность независимо от devicePixelRatio.

### PriceLevelPlugin

**Назначение:** отображение одной горизонтальной линии, её ценовой метки, drag-and-drop и удаления.

**Механика:** 1. **Инициализация:** при attached() вычисляет начальную Y-координату через series.priceToCoordinate(price). 2. **Основной canvas (paneViews):** - Горизонтальная линия #7b61ff толщиной 1 px на всю видимую ширину графика. - При наведении (isHovered) или перетаскивании (isDragging) рисуется полупрозрачная фиолетовая полоса rgba(123, 97, 255, 0.2) высотой **16 px** (8 px в каждую сторону от линии). - В правой части линии появляется красный круг (#ef5350) с белым крестиком - кнопка удаления. 3. **Canvas шкалы цены (priceAxisPaneViews):** - Фиолетовый прямоугольник с текущей ценой (toFixed(2)), привязанный к Y-координате линии. 4. **Drag-and-drop:** - mousedown внутри полосы (±8 px от линии) активирует \_isDragging = true. - Глобальный mousemove (на window) переводит координату курсора обратно в цену через coordinateToPrice(). - Цена обновляется в реальном времени, метка на шкале и линия перерисовываются через requestUpdate(). - mouseup сбрасывает флаг \_isDragging. 5. **Удаление:** - mousedown в правой части (зона кнопки «X») вызывает onRemove(id). - detached() удаляет все глобальные и локальные слушатели событий.

## Детали реализации

### Координатные пространства

lightweight-charts v5 использует **bitmap coordinate space** с учётом devicePixelRatio (DPR). Все renderer'ы работают через target.useBitmapCoordinateSpace(scope), где:

- scope.horizontalPixelRatio - масштаб по X (DPR).
- scope.verticalPixelRatio - масштаб по Y (DPR).

Все размеры (кнопка 24 px, полоса 8 px, шрифт 11 px) умножаются на соответствующий pixel ratio, что гарантирует чёткость на Retina-экранах.

### Генерация данных

data.js создаёт случайное блуждание (random walk) с волатильностью 2%:

close = open + change  
high = max(open, close) + |change| \* random(0..0.5)  
low = min(open, close) - |change| \* random(0..0.5)

- Интервал: 1 день (setUTCDate + 1).
- Защита: low ограничен снизу Math.max(0, low).
- Начальная цена: \$50 000.

### Жизненный цикл линий

Пользователь кликает кнопку "+"  
→ main.js создаёт new PriceLevelPlugin(id, price, onRemove)  
→ candlestickSeries.attachPrimitive(plugin)  
→ plugin.attached() подписывается на события  
→ plugin рисуется на двух canvas (график + шкала цены)  
<br/>Пользователь удаляет линию  
→ onRemove(id) вызывает detachPrimitive()  
→ plugin.detached() очищает слушатели  
→ plugin удаляется из массива priceLevels

### Производительность

- Каждая линия - отдельный инстанс плагина. Это соответствует архитектуре lightweight-charts и позволяет легко добавлять/удалять линии без перерисовки всего графика.
- requestUpdate() вызывается только при изменении состояния (появление кнопки, движение курсора, drag), а не на каждый кадр.

## Известные ограничения

- **Масштабирование:** при сильном зуме графика ценовая метка на шкале может наслаиваться на соседние метки библиотеки. Это ограничение canvas-рендеринга; для production-решений рекомендуется добавить проверку минимального расстояния между метками.
- **Количество линий:** мягкий лимит ~50 линий из-за перерисовки canvas. В рамках ТЗ (10-20 линий) производительность оптимальна.
- **Touch-события:** текущая реализация оптимизирована для десктопа (mouse events). Для мобильных устройств требуется добавить обработчики touchstart / touchmove / touchend.
- **Сохранение состояния:** созданные линии не сохраняются между перезагрузками страницы (не требовалось по ТЗ).
